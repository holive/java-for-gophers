# Resources

For days when exercises feel too heavy.

Reading or watching counts as learning. You don't have to write code every session.

---

## Books

### Start Here

**Effective Java** (3rd Edition) by Joshua Bloch
- The most useful book for understanding Java idioms
- Skip around. Recommended items for Gophers:
  - Item 1: Static factories vs constructors
  - Items 10-14: equals, hashCode, toString, Comparable
  - Item 17: Minimize mutability (you'll like this one)
  - Items 57-64: General programming best practices
- Why it helps: Shows *why* Java does things differently than you expect

### When You're Ready

**Spring Boot in Action** by Craig Walls
- Explains the magic behind Spring annotations
- Good for when you want to understand, not just use
- Skip if you prefer learning by doing

**Modern Java in Action** by Urma, Fusco, Mycroft
- Deep dive on streams, lambdas, Optional
- Useful if you want to write more functional Java
- Skip if imperative style is fine for now

### Deep Dives (Optional)

**Java Concurrency in Practice** by Brian Goetz
- When you hit real threading problems
- Dense. Not for casual reading.
- Wait until you actually need it

**Clean Code** by Robert Martin
- If you want philosophy about code quality
- Some parts are dated, but core ideas hold
- Good for: understanding why Java devs write the way they do

## Online (Free)

### Quick Reference

**Baeldung.com**
- Searchable Java/Spring how-tos
- Quality varies, but usually solid
- Good for: "how do I do X in Spring Boot?"

**Spring.io Guides**
- Official tutorials, brief and focused
- https://spring.io/guides
- Good for: step-by-step walkthroughs of specific features

**Java Design Patterns**
- https://java-design-patterns.com
- Visual pattern explanations with code
- Good for: understanding common Java patterns

### Documentation

**Spring Boot Reference**
- https://docs.spring.io/spring-boot/docs/current/reference/html/
- Comprehensive but dense
- Good for: looking up specific configuration options

**Spring Cloud AWS**
- https://docs.awspring.io/spring-cloud-aws/docs/current/reference/html/
- Your company's stack
- Good for: SQS, SNS, S3, DynamoDB integration

### Videos

**Amigoscode (YouTube)**
- Beginner-friendly Spring Boot tutorials
- Long-form, but easy to follow
- Good for: passive learning when you can't focus on text

**Spring I/O Conference Talks**
- https://www.youtube.com/c/SpringSourceDev
- More advanced, architecture-focused
- Good for: understanding *why* Spring does things

**Java Brains (YouTube)**
- Clear explanations of Java/Spring concepts
- Good pacing for experienced developers
- Good for: filling specific knowledge gaps

## Reading Strategy

### Low Energy
- Scan Baeldung for a topic you already know
- Watch a video at 1.5x speed
- Read one item from Effective Java

### Medium Energy
- Follow a Spring.io guide end-to-end
- Read a chapter of Spring Boot in Action
- Watch a conference talk

### High Energy
- Do exercises instead of reading
- Reading is for recovery days

---

Consuming content is still progress. Some days, that's enough.
